Document Your MVP for a Developer.

MVP 를 만들기 전에 준비해 봐야 하는 것들. 어차피 나 스스로 개발할 꺼면 꼭 필요하지 않을 수도 있지만, 계획을 세워놔야 집중이 유지되는 편이라 짧은 시간을 마련해서라도 문서화를 간략히 해두는 걸 좋아한다. 아래 문서를 만들 때 특별한 형식은 없다. 편안하게 노트에 써놔도 상관 없을 것 이다.

  • Business Concept : 비지니스를 보다 폭넓게 이해하기 위한 자료. 린캔버스 같은 것을 말한다.
  • Customer Development Notes : 고객 인터뷰를 하기 위한 자료.
  • Prioritized User Stories : 고객의 문제를 정의하고 사용자 스토리 기반으로 시작하라. 우선순위를 메겨라.
  • Product Feature List : 기능 목록을 작성하라.
  • Functional Details : 우선순위가 높은 기능들에 대해서 상세 명세를 작성하라.
  • Wireframes, Comps, Clickable Prototype : 와이어프레임 프로토타입이나, 스케치를 만들어라.
  • Other Documentation : Personas, Use cases, Software Requirements.
