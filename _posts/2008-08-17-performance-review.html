퍼포먼스 리뷰를 했습니다. 다음 한 해의 연봉을 결정하고 지난 한 해의 성적표를 받는 날입니다. 이번 해는 나름 열심히 했다고 생각했는데 결과가 썩 마음에 들지 않네요. 하루 정도 우울해 하긴 했지만 이대로 손놓고 있을 수는 없기에 대책을 찾아보기로 했습니다.

회사에서 평가시 사용하는 항목에 대해 제가 취할 수 있는 액션 아이템들을 정리해 봅니다.

Action item Category Etc
유명 개발자들의 dev plan 을 읽어보고 좋은 스타일을 모방해보자 Design  
파트너의 이슈에 관심을 가지고 도와주자 Design, Customer/partner  
사내 교육 자료를 활용하고 공부하자 Design  
일을 하면서 생각나는 아이디어들을 따로 메모해두자 Design  
SWAG, ETA, daily schedule 순으로 정밀도를 높여 스케쥴링한다. 이것을 팀원과 공유하자 Implementation  
스케쥴을 짜기 전 dev plan 을 조금이라도 쓴 다음 계획하자 Implementation  
PM 의 Spec 리뷰에 적극 참여하고 빠른 답변을 주자 Implementation  
새로운 개발 practice 를 직접 시도해보자 Implementation  
unit test 를 작성해 보자 Quality  
Perl 공부를 시작하자 Quality  
버그는 보이는 즉시 리포트 하자 Quality  
팀원들의 코드 리뷰에 응하자 Sound engineering practices  
팀원들의 dev plan document 리뷰에 참여하자 Sound engineering practices  
신입 사원의 질문에 적극적으로 대답해주자 Leadership  

일단 이메일 답장을 정성스레 쓰는 것 부터 시작해야 겠군요.ㅋ
