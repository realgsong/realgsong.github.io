신개념 N-Screen 웹앱 프레임워크 PARS

[slideshare id=39615151&w=427&h=356&style=border:1px solid #CCC; border-width:1px; margin-bottom:5px; max-width: 100%;&sc=no]

[1D1]신개념 N스크린 웹 앱 프레임워크 PARS from NAVER D2

첫 세션이다 보니 제일 맑은 정신으로 들었다. 시작은 N-Screen 의 발전과정에 대한 소개 부터..

  1. Fixed Service
  2. Cooperating Applications
  3. Cooperating Apps with Launch
  4. A Distributed Application

PARS 는 이 중에서 제일 마지막 distributed application 그 중에서도 web app 을 위한 프레임워크다. 쉽게 이야기하자면, 스마트폰에 실행중인 어플리케이션을 티비에서 동일한 상태로 실행해 볼 수 있다는 개념이다. 어플리케이션의 화면을 스트리밍 하는게 아니라, 실행 컨텍스트에 대한 것도 모두 넘겨줘서, 원격지에 있는 객체가 해당 이벤트들을 처리하도록 하는 것.

이와 유사한 Presentation API 라는 걸 웹표준으로 추진하고 있다던데, 기대된다. 관련자료를 찾아볼 필요가 있다.

WebRTC 개발, 현재 그리고 미래

[slideshare id=39609277&w=427&h=356&style=border:1px solid #CCC; border-width:1px; margin-bottom:5px; max-width: 100%;&sc=no]

[1C2]webrtc 개발, 현재와 미래 from NAVER D2

아자르 서비스에 대한 소개로 시작하여 WebRTC 를 쓰면, 쉽게 화상 채팅 솔루션 구현이 가능하다. 라고 결론을 내리는 줄 알았으나, 진짜 결론은 제품화하려면 그걸로는 안되지롱으로 끝났음. RTC 쪽이 다양한 아이디어에서 활용가능한 부분이 있을 것 같다. 최적화 부분에서는 운영하면서 조금씩 가다듬어 가는 것 말고 방법이 있겠는가? 어떤 서비스이고 어떤 컨텐츠가 오가느냐에 따라서 실시간 채팅의 퍼포먼스 목적도 달라질테니 말이다.

코덱에 따라 브라우저의 관련 소스를 보고 작업해야 하는 애로사항들이 있다고 한다.

지금까지 상상한 표현의 한계를 넘자, webGL

[1A3]지금까지 상상한 표현의 한계를 넘자 WebGL from NAVER D2

모바일 브라우저에서 3d 그래픽을 표시해야 한다면, webGL 이 정답이다. GPU 를 활용해서 좋은 퍼포먼스를 보여준다. 안드로이드 4.x 이상 ios 8 이상 기본 탑재된다고 한다. 예전 회사에서 미디어센터 ui 작업했던 게 생각나면서, 그때 이걸 도입해서 진행했으면 input 대비 결과가 조금 더 좋아졌을까? 하는 생각을 잠깐 했다.

발표가 1,2 부로 나눠지는데, 1부에 쓰인 슬라이드 참 화려하게 잘 만들었음.

[1C5]Lessons from developing a web browser for raspberry pi from NAVER D2

발표자가 WebKit commiter 라고 해서, 우와~ 함. 라즈베리파이 같은 제한된 환경에서 브라우저를 만들면서 고생한 경험담. 세부적인 데테일들은 잘 모르니 그러려니하고 넘겼다. 다른 사람이 고생한 이야기를 듣는 건 항상 재밌는 일.

오픈소스 하드웨어 플랫폼과 Node.js로 구현하는 IoT 플랫폼

[slideshare id=39637830&w=427&h=356&style=border:1px solid #CCC; border-width:1px; margin-bottom:5px; max-width: 100%;&sc=no]

[1C6]오픈소스 하드웨어 플랫폼과 Node.js로 구현하는 IoT 플랫폼 from NAVER D2

아두이노나 라즈베리 같은데서 node.js 돌려서 이것저것 만들었다는 이야기. IOT 이야기와 node.js 가 임베디드 쪽에서의 활용도에 대한 것들로 관심이 많았으나, 감기 몸살로 방전 상태에서 들었다. 매직에코에서 만든 루미스마트라는 제품이 재미있었다.

데뷰 같은 컨퍼런스에 오면, 여러모로 다양한 각도의 자극을 받을 수 있어서 좋다. 생각 중인 아이디어들과 합쳐 지기도 하고, 몰랐던 시장과 기술을 알 수 있게 되어서 좋기도 하다. 몸살만 아니었으면, 마지막 세션까지 듣고 가려고 했는데 안타깝다.
